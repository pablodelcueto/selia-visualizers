# read(startIndexopt, startTimeopt, endIndexopt, endTimeopt, durationIndexopt, durationTimeopt, channelopt) → {module:Audio/audioFile.audioData}

Extracts audio data from buffer.

Main method for data extraction. Data can be requested in several ways:

  1. If no argument is given, the all currently loaded data will be returned.
  2. If startIndex or startTime are provided, the returned data will only contain audio samples later than startTime or the time corresponding to startIndex, see getWavIndexFromTime. If both are given at the same time only startTime will be used.
  3. If endTime or endIndex are provided, the returned data will only contain audio samples before than endTime or the time corresponding to endIndex. If both are given at the same time only endTime will be used.
  4. If durationTime or durationIndex are provided, the returned data will only contain audio samples contained in the interval [startTime, startTime + durationTime], where startTime is either the default value or user specified (as in point 2). DurationIndex will be translated into time if provided. If both are given at the same time only durationTime will be used.
  5. If channel argument is provided the resulting data will be read from the desired channel. Otherwise it will default to the first channel.

The returned data will contain all loaded values within the specified interval, but it may return a fraction of the requested data if loading is not complete. The bounds of the returned data are specified in the returned object.

First buffer index in requested slice.

Initial time in requested slice. Alternative to startIndex.
endIndex number <optional>

Final buffer index in requested slice.

Final time in requested slice. Alternative to endIndex.
durationIndex number <optional>

Slice length of requested data.
durationTime number <optional>

Time length of requested data. Alternative to durationIndex.
channel number <optional>

Requested audio channel.
